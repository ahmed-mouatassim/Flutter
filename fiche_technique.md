# Fiche Technique — تطبيق حجز المرافق الرياضية لمؤسسة إبداع


# 1. نظرة عامة /Résumé

**اسم المشروع:** Ibdaa Sports Booking (مؤقت)

**الغرض:** تطبيق مخصص لمتدربي مؤسسة إبداع لحجز مرافق رياضية (ملعب كرة القدم + ملعبين للتنس).

**المستخدمون المستهدفون:** متدربو المؤسسة (Users) وطاقم الإدارة (Admin).

**منصة التطوير:** تطبيق موبايل (Android / iOS) مبني بـ **Dart + Flutter** للواجهة الأمامية، وللـ Backend: **PHP + MySQL** أو **Firebase** .



# 2. أهداف ومزايا التطبيق (Requirements — Functional)

## 2.1 حسابات المستخدمين

* نوعا الحساب: **متدرب** و **إدارة**.
* عند إنشاء حساب متدرب: يبقى الوضع `pending` حتى توافق الإدارة.
* بيانات التسجيل المطلوبة:

  * الاسم والنسب (first_name, last_name)
  * الإيميل
  * كلمة المرور
  * الدورة التدريبية الخاصة به
  * المجمع المنخرط فيه (مديونة، تيط مليل، الهرويين)

## 2.2 مزايا حساب المتدرب

* حجز الملعب (كرة قدم أو تنس) — اختيار الملعب، التاريخ، الساعة.
* إنشاء مباراة (Match) وفتح المشاركة فيها.
* الانخراط (Join) في مباريات منشأة من طرف متدربين.
* عرض صفحة المباريات القادمة (مع الوقت، اللاعبون، زر الانخراط والمشاركة).
* صفحة المناسبات العامة.
* صفحة معلومات المستخدم (عرض وتحديث بعض الحقول).
* مشاركة رابط المباراة مع الأصدقاء (مشاركة نظامية في الهاتف).

## 2.3 مزايا حساب الإدارة

* قبول/رفض حسابات المتدربين (pending → accepted/rejected).
* قبول/رفض مباريات المنشأة من قبل المتدربين (إذا أردت اعتماد مسبق للمباريات).
* إلغاء أيام/ساعات (وضع غير متاح للحجز)، وإدارة جدول صيانة/إجازات الملعب.
* رؤية كل الحجوزات والمباريات وتقاريرها.


# 3. صفحات (Screens) وواجهات المستخدم UI :

### المكونات الأساسية:

## شاشة الترحيب / Splash

* شعار المؤسسة وإجراء تحقق لتوكن المستخدم.
* إعادة توجيه إلى شاشة الدخول أو Home حسب حالة الجلسة.

##  شاشة التسجيل (Register)

* حقول: الاسم، النسب، الإيميل، كلمة المرور، تأكيد كلمة المرور، الدورات (select / chips)، المجمع (dropdown).
* زر: "إنشاء حساب" → تعرض رسالة: **"حسابك قيد الانتظار حتى توافق الإدارة"**.

##  شاشة الدخول (Login)

* إيميل، كلمة المرور، زر تسجيل.
* رسالة واضحة للحالة: pending / rejected / accepted.

##  داشبورد المتدرب (Home)

* رُبط سريع: حجز ملعب جديد، المباريات القادمة، المناسبات، ملف المستخدم.
* بطاقة الحالة: "حالة الحساب: مقبول/قيد الانتظار".

##  صفحة الحجز (New Booking)

* اختيار الملعب ، اختيار التاريخ ، اختيار الوقت ، زر تحقق من التوفر → تأكيد الحجز (ينشأ بحالة pending أو accepted حسب سياسة الإدارة).

##  صفحة المباريات (Matches)

* قائمة المباريات مع: التاريخ، الوقت، الملعب، عدد المشاركين، زر Join / Leave، زر Share.
* صفحة تفاصيل المباراة مع قائمة اللاعبين المسجلين وزر الانخراط.

##  صفحة المناسبات (Events)

* مناسبات المؤسسة (آلية عرض بسيطة — CRUD من طرف Admin).

##  صفحة الملف الشخصي (Profile)

* عرض وتحديث البيانات: الاسم، الدورات، المجمع، تغيير كلمة المرور.

##  واجهة الإدارة (Admin Panel — داخل نفس التطبيق بحساب admin)

* تبويب الحسابات المعلقة (قوائم مع أزرار قبول/رفض).
* تبويب الحجوزات (قائمة، فلترة حسب الملعب والتاريخ، قبول/رفض).
* تبويب المباريات (قبول/رفض).
* واجهة لإلغاء أيام/ساعات (blocked slots).
